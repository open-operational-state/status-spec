# Terminology

This directory captures spec-level applied terminology — how terms from the [authoritative glossary](https://github.com/open-operational-state/governance/blob/main/GLOSSARY.md) are used in specification context, with additional nuance, examples, and usage guidance.

## Relationship to Glossary

The **governance glossary** (`governance/GLOSSARY.md`) is the authoritative source for controlled terminology definitions.

This directory provides:

- **Applied context** — how terms are used specifically within the specification
- **Usage examples** — illustrative examples of concepts in practice
- **Disambiguation** — clarifying distinctions that matter in spec context
- **Domain-specific nuance** — additional precision needed for normative text

## Status

This directory will be populated as the specification develops and terminology needs arise. It is intentionally empty during the scaffolding phase.
