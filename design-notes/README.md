# Design Notes

This directory contains design rationale, working notes, and exploratory analysis that inform the specification but are not themselves normative.

## Purpose

Design notes capture:

- Rationale for significant design decisions
- Trade-off analysis for architectural choices
- Exploratory work on specific topics
- Background research and findings

## Relationship to ADRs

Governance-level Architecture Decision Records (ADRs) live in [`governance/decisions/`](https://github.com/open-operational-state/governance/blob/main/decisions/). Design notes here are more technical and spec-focused.
