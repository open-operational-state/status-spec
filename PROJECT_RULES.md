## Repo-Specific Constraints

- This repository contains **markdown only**. No TypeScript, no code.
- All content is licensed under **CC BY 4.0**.
- The six-layer architecture document (`ARCHITECTURE.md`) must be written and stabilized before detailed spec documents are drafted.
- Do not define normative JSON field names or wire schemas until architecture, scope, and terminology are disciplined.
- Terminology in this repository (`terminology/`) captures applied usage and context. The authoritative glossary lives in the `governance` repository.
- Prior art drafts are influences and compatibility targets, not normative foundations.
